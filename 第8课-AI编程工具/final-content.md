## Section 1：AI 编程工具的定位（15 分钟）

### 1.1 重新认识 AI 编程工具

首先，我们要重新认识 AI 编程工具。很多人觉得 AI 编程工具就是代码补全，就是 GitHub Copilot 那样的东西。这个认知是不完整的。

我给大家看一个对比：

**传统代码补全（如 IntelliSense）：**
- 基于语法分析和类型推断
- 只能补全当前文件的上下文
- 没有语义理解能力
- 不能跨文件推理

**AI 代码补全（如 GitHub Copilot）：**
- 基于大语言模型
- 可以理解注释和函数名的语义
- 能生成整段代码
- 但仍然是"单向输出"，你不能和它对话

**AI 结对编程（如 Cursor、Windsurf）：**
- 不仅能生成代码，还能理解你的意图
- 可以跨多个文件进行编辑
- 能够对话、解释、重构
- 具备项目级别的上下文理解

这就是我们今天要重点讲的：AI 结对编程。

### 1.2 什么是 AI 结对编程？

传统的结对编程（Pair Programming）是两个程序员坐在一起，一个人写代码（Driver），另一个人审查和思考（Navigator）。AI 结对编程也是类似的概念，只不过你的搭档是 AI。

但这里有个关键点：AI 不是来替代你的，而是来增强你的。你仍然是主导者，AI 是辅助者。

我给大家举个例子。假设你要重构一个组件，把它从 Class Component 改成 Function Component。

**传统方式：**
1. 你手动改写组件
2. 你手动更新所有引用
3. 你手动测试
4. 你手动修复 bug

**AI 结对编程方式：**
1. 你告诉 AI："把这个组件改成 Function Component"
2. AI 自动改写组件，自动更新引用，甚至帮你写测试
3. 你审查 AI 的修改，提出调整意见
4. AI 根据你的反馈继续优化

看到区别了吗？你的角色从"执行者"变成了"决策者"。你把精力放在思考架构、设计模式、业务逻辑上，而不是敲键盘上。

### 1.3 AI 编程工具的三个层次

我把 AI 编程工具分为三个层次：

**第一层：代码生成**
- 根据注释生成代码
- 根据函数名生成实现
- 这是最基础的功能，所有 AI 工具都有

**第二层：上下文感知**
- 理解当前文件的上下文
- 理解项目的依赖关系
- 能够跨文件推理
- Cursor、Windsurf 都在这个层次

**第三层：Memory 驱动**
- 记住你的编码习惯
- 记住项目的架构规范
- 记住你的偏好和风格
- 这是我们今天要重点讲的

大多数人只用到了第一层，少数人用到了第二层，而真正的高手会用到第三层。

好，现在大家对 AI 编程工具有了一个全新的认识。接下来，我们深入到 Cursor 这个工具，看看它到底有多强大。

## Section 2：Cursor 深度解析（40 分钟）

### 2.1 Cursor 的三大核心功能

Cursor 有三个核心功能，我称之为"三板斧"：

1. **Composer**：多文件编辑，适合大规模重构
2. **Chat**：上下文感知对话，适合问答和探索
3. **Cmd+K**：行内编辑，适合快速修改

这三个功能各有各的用途，不能互相替代。很多人只用 Chat，这就浪费了 Cursor 的 80% 功能。

### 2.2 Composer：多文件编辑的艺术

Composer 是 Cursor 最强大的功能，但也是最容易被忽视的功能。

**什么时候用 Composer？**
- 需要同时修改多个文件
- 需要创建新的功能模块
- 需要重构代码架构
- 需要批量更新代码风格

我给大家演示一个真实场景。假设我们要给项目添加一个"用户认证"功能，涉及到多个文件：

```
src/
  lib/
    auth.ts          # 认证逻辑
  app/
    api/
      auth/
        route.ts     # API 路由
    login/
      page.tsx       # 登录页面
  components/
    LoginForm.tsx    # 登录表单
  middleware.ts      # 中间件
```

如果用传统方式，你需要一个文件一个文件地创建和编辑。但用 Composer，你只需要这样：

**步骤 1：打开 Composer（Cmd+I 或 Ctrl+I）**

**步骤 2：输入指令：**
```
