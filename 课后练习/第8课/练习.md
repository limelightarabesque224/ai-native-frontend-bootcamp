# 第 8 课练习：AI 编程工具

## 练习 1：Cursor Composer 模式（基础）

**目标**：使用 Cursor 的 Composer 模式完成一个功能。

**任务**：

用 Cursor Composer 为 `demo/` 项目添加一个 "主题切换" 功能：

1. 打开 Cursor Composer（Cmd+I）
2. 输入 Prompt：
   ```
   Add a dark mode toggle to the demo project.
   - Use Zustand for state management
   - Add a toggle button in the header
   - Persist theme preference in localStorage
   - Update Tailwind config for dark mode
   ```

3. 让 AI 生成完整的实现
4. Review 代码并测试

**验收标准**：
- [ ] 使用 Cursor Composer 生成代码
- [ ] 主题切换功能正常工作
- [ ] 刷新页面后主题保持
- [ ] 记录 Prompt 和生成过程

---

## 练习 2：Memory 管理（进阶）

**目标**：为项目配置 AI Memory，提升 AI 理解项目的能力。

**任务**：

创建项目级 Memory 文件：

1. **项目 Memory** (`.claude/memory/project.md`)
   - 项目技术栈
   - 代码规范
   - 常用模式

2. **组件 Memory** (`.claude/memory/components.md`)
   - shadcn/ui 组件使用规范
   - 自定义组件模板
   - 样式约定

3. **测试 Memory**
   - 用 AI 工具生成一个新组件
   - 验证 AI 是否遵循了 Memory 中的规范

**验收标准**：
- [ ] 创建至少 2 个 Memory 文件
- [ ] Memory 内容结构清晰
- [ ] AI 生成的代码符合规范
- [ ] 对比有/无 Memory 的生成质量

## 练习 3：.cursorrules 优化（挑战）

**目标**：编写一套完整的 .cursorrules，让 AI 生成的代码质量最大化。

**任务**：

优化 `demo/.cursorrules`，包含：

1. **技术栈约束**
   - 必须使用的库和版本
   - 禁止使用的库

2. **代码风格**
   - 命名规范
   - 文件组织
   - 注释规范

3. **最佳实践**
   - 性能优化
   - 可访问性
   - 错误处理

4. **AI 生成指令**
   - 优先使用现有组件
   - 遵循项目模式
   - 生成完整的类型定义

测试方法：
- 用 AI 生成 3 个不同的组件
- 验证是否都遵循了 .cursorrules
- 记录违规情况并优化规则

**验收标准**：
- [ ] .cursorrules 至少 50 行
- [ ] 覆盖所有关键规范
- [ ] AI 生成的代码一致性 > 90%
- [ ] 包含实际测试案例
