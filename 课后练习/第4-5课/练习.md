# 第 4-5 课练习：Design to Code

## 练习 1：Figma to Code（基础）

**目标**：使用 Figma AI 或 Penpot 将设计稿转换为代码。

**任务**：

1. 在 Figma Community 找一个简单的 Landing Page 设计
2. 使用以下工具之一转换为代码：
   - Figma Dev Mode + AI
   - Penpot（开源替代）
   - Pencil（Figma 插件）

3. 将生成的代码迁移到 `demo/` 项目
4. 用 Tailwind CSS 重构样式

**验收标准**：
- [ ] 记录使用的工具和步骤
- [ ] 生成的代码可运行
- [ ] 样式迁移到 Tailwind
- [ ] 视觉还原度 > 90%

---

## 练习 2：v0.dev 生成组件（进阶）

**目标**：使用 v0.dev 生成复杂组件并集成到项目。

**任务**：

用 v0.dev 生成以下组件：

1. **定价表**（Pricing Table）
   - 3 个定价方案
   - 月付/年付切换
   - 功能对比列表
   - CTA 按钮

2. **功能特性区域**（Features Section）
   - 6 个特性卡片
   - 图标 + 标题 + 描述
   - 响应式网格布局

Prompt 示例：
```
Create a pricing table with 3 tiers (Basic, Pro, Enterprise).
Include monthly/yearly toggle, feature comparison, and CTA buttons.
Use shadcn/ui components and Tailwind CSS.
```

**验收标准**：
- [ ] 记录完整的 Prompt
- [ ] 生成的代码集成到 `demo/`
- [ ] 根据需求调整了代码
- [ ] 组件可复用

## 练习 3：Bolt.new 全栈应用（挑战）

**目标**：使用 Bolt.new 生成一个全栈应用原型。

**任务**：

用 Bolt.new 生成一个 "待办事项应用"：

功能需求：
- 添加/删除/编辑任务
- 标记完成状态
- 按状态筛选（全部/进行中/已完成）
- 数据持久化（localStorage）
- 响应式设计

技术栈：
- Next.js 15 + TypeScript
- shadcn/ui
- Tailwind CSS

**验收标准**：
- [ ] 使用 Bolt.new 生成初始代码
- [ ] 所有功能正常工作
- [ ] 代码质量良好（无明显 bug）
- [ ] 记录生成过程和修改记录
- [ ] 对比 AI 生成 vs 手写的效率差异
