## 练习 2：Next.js Server Actions（进阶）

**目标**：使用 Server Actions 实现表单提交。

**任务**：

将 `demo/src/app/demos/form/page.tsx` 改为使用 Server Actions：

1. 创建 Server Action：
   ```ts
   // app/actions/user.ts
   'use server'

   import { z } from 'zod'

   const schema = z.object({
     name: z.string().min(2),
     email: z.string().email(),
   })

   export async function createUser(formData: FormData) {
     const data = schema.parse({
       name: formData.get('name'),
       email: formData.get('email'),
     })

     // 保存到数据库
     await db.user.create({ data })

     return { success: true }
   }
   ```

2. 在表单中使用：
   ```tsx
   <form action={createUser}>
     <input name="name" />
     <input name="email" />
     <button type="submit">提交</button>
   </form>
   ```

3. 添加 `useFormStatus` 显示加载状态

**验收标准**：
- [ ] 使用 Server Actions
- [ ] 表单验证在服务端
- [ ] 显示加载状态
- [ ] 错误处理完善

## 练习 3：全栈 CRUD 应用（挑战）

**目标**：用 Next.js + Prisma + tRPC 构建一个全栈应用。

**任务**：

创建一个 "博客管理系统"：

1. **技术栈**
   - Next.js 15 Server Actions
   - Prisma（SQLite）
   - tRPC（类型安全 API）
   - shadcn/ui

2. **功能**
   - 文章列表（分页）
   - 创建文章
   - 编辑文章
   - 删除文章
   - 搜索文章

3. **数据库 Schema**
   ```prisma
   model Post {
     id        String   @id @default(cuid())
     title     String
     content   String
     published Boolean  @default(false)
     createdAt DateTime @default(now())
     updatedAt DateTime @updatedAt
   }
   ```

4. **tRPC Router**
   ```ts
   export const postRouter = router({
     list: publicProcedure
       .input(z.object({ page: z.number() }))
       .query(({ input }) => {
         return db.post.findMany({
           skip: input.page * 10,
           take: 10,
         })
       }),
     create: publicProcedure
       .input(z.object({ title: z.string(), content: z.string() }))
       .mutation(({ input }) => {
         return db.post.create({ data: input })
       }),
   })
   ```

**验收标准**：
- [ ] 完整的 CRUD 功能
- [ ] 类型安全（前后端共享类型）
- [ ] 数据持久化
- [ ] 响应式 UI
- [ ] 错误处理和加载状态
- [ ] 使用 Biome 检查代码质量
