# 第 3 课练习：Radix UI

## 练习 1：构建 Dropdown Menu（基础）

**目标**：使用 Radix UI 原语构建一个用户菜单。

**任务**：

使用 `@radix-ui/react-dropdown-menu` 创建一个用户头像下拉菜单：

```
[头像] ▼
  ├─ 个人资料
  ├─ 设置
  ├─ ─────────  (分隔线)
  ├─ 退出登录
```

要求：
- 点击头像打开菜单
- 包含图标（使用 lucide-react）
- 包含快捷键提示（如 ⌘K）
- 包含分隔线

**验收标准**：
- [ ] 使用 Radix Dropdown 原语
- [ ] 键盘导航正常（方向键、Enter、Esc）
- [ ] 包含 ARIA 属性
- [ ] 样式使用 Tailwind

## 练习 2：构建 Popover（进阶）

**目标**：用 Radix Popover 构建一个日期选择器。

**任务**：

使用 `@radix-ui/react-popover` 创建一个简单的日期选择器：
- 点击输入框弹出日期面板
- 显示当前月份的日历
- 点击日期后关闭 Popover 并填充输入框
- 点击外部区域关闭

**验收标准**：
- [ ] 使用 Radix Popover 原语
- [ ] 焦点管理正确
- [ ] 支持 Esc 关闭
- [ ] 位置自动调整（避免溢出屏幕）

## 练习 3：构建 Accessible Modal（挑战）

**目标**：用 Radix Dialog 构建一个完全可访问的确认对话框。

**任务**：

创建一个 "删除确认" 对话框，满足 WCAG 2.1 AA 标准：

1. 使用 `@radix-ui/react-dialog`
2. 包含：
   - 标题（Dialog.Title）
   - 描述（Dialog.Description）
   - 危险操作警告
   - 取消/确认按钮
3. 可访问性要求：
   - 焦点陷阱（打开后焦点在对话框内循环）
   - 初始焦点在 "取消" 按钮
   - Esc 关闭
   - 背景滚动锁定
   - 屏幕阅读器友好

**验收标准**：
- [ ] 使用 Radix Dialog 所有关键原语
- [ ] 焦点管理符合 ARIA 规范
- [ ] 用屏幕阅读器测试（VoiceOver/NVDA）
- [ ] 键盘导航完整
- [ ] 包含动画（进入/退出）
