## 报告漏洞

如果你发现了安全漏洞，请**不要**公开提交 Issue。

请通过以下方式报告：

1. 发送邮件至：security@example.com
2. 在邮件中包含：
   - 漏洞的详细描述
   - 复现步骤
   - 潜在影响
   - 可能的修复建议（如果有）

我们会在 48 小时内回复你的报告，并在确认漏洞后尽快发布修复。

## 安全最佳实践

### 环境变量

- 永远不要将 `.env` 文件提交到版本控制
- 使用 `.env.example` 作为模板
- 在生产环境中使用环境变量管理服务

### API 密钥

- 不要在客户端代码中暴露 API 密钥
- 使用服务端 API 路由处理敏感操作
- 定期轮换 API 密钥

### 依赖管理

- 定期运行 `pnpm audit` 检查依赖漏洞
- 及时更新有安全问题的依赖
- 使用 `pnpm-lock.yaml` 锁定依赖版本

### 代码审查

- 所有代码变更都需要经过审查
- 特别关注用户输入处理和数据验证
- 避免 XSS、SQL 注入等常见漏洞
