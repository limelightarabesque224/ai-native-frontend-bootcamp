# 大模型时代的前端开发 - 技术演讲大纲

**演讲主题**: 大模型时代的前端开发：AI-Native 技术栈与工作流

**目标受众**: 具备项目与工程化经验的前端工程师

**演讲时长**: 每节课 2-2.5 小时（共 12 节课，含第 0 课）

**核心观点**: 在大模型时代，前端技术选型需要增加新的评估维度——AI 友好性。本课程不是教"如何用 AI 写代码"，而是教"如何构建 AI-Native 的前端技术栈和工作流"。

## 第 0 课：认知重构 - AI 友好性是新的选型维度

### 核心观点
传统技术选型的三大维度（DX、性能、生态）已经不够，需要增加第四个维度：AI 友好性。

### 关键要点
1. **LLM 如何理解代码**
   - Token 化机制
   - 上下文窗口限制
   - 代码可读性对 AI 的影响

2. **什么是 AI 友好的代码**
   - 语义内联（Semantic Inline）
   - 源码可见（Source Visible）
   - 约定优于配置（Convention over Configuration）
   - 组合优于继承（Composition over Inheritance）

3. **技术地图预览**
   - 样式层：Tailwind CSS v4
   - 组件层：shadcn/ui + Radix UI
   - 工具层：Figma AI、v0.dev、Bolt.new
   - 测试层：Playwright MCP
   - 架构层：Monorepo
   - 开发工具：Cursor、Windsurf
   - AI 集成：Vercel AI SDK

## 第 1 课：样式方案革命 - Tailwind CSS v4

### 核心观点
Utility-first CSS 是 AI 最佳拍档，Tailwind v4 带来了性能革命。

### 关键要点
1. **传统 CSS 方案的 AI 盲区**
   - BEM：语义分散
   - CSS Modules：作用域隔离
   - styled-components：运行时开销

2. **Tailwind v4 核心变化**
   - CSS-first 配置（@theme、@plugin）
   - Oxide 引擎（10x 性能提升）
   - 新颜色系统（v4.2+）

3. **为什么 utility-first 是 AI 最佳拍档**
   - 语义在 HTML 中内联
   - 原子化类名，AI 可精确组合
   - 无需跨文件推理

4. **横向对比**
   - UnoCSS、CSS Modules、styled-components 对比表

### 演示案例
用 AI 工具生成复杂布局，对比不同 CSS 方案的效率

## 第 2 课：组件库范式转移 - shadcn/ui

### 核心观点
"源码可见"比"npm 黑盒"更 AI 友好，Copy-Paste 模式是未来。

### 关键要点
1. **传统组件库的 AI 困境**
   - npm 黑盒：AI 看不到源码
   - 过度封装：AI 难以定制
   - 版本锁定：升级困难

2. **shadcn/ui 的 Copy-Paste 哲学**
   - 不是 npm 包，是代码片段
   - CLI 工作流
   - Registry 系统

3. **shadcn/ui 生态**
   - magic-ui（动画）
   - aceternity-ui（现代 UI）
   - TweakCN（AI 主题编辑器）
   - v0.dev（AI 生成）

4. **横向对比**
   - Ant Design、MUI、Chakra UI、Park UI 对比表

## 第 3 课：无头组件的底层逻辑 - Radix UI

### 核心观点
Headless UI 通过分离逻辑与样式，让 AI 专注于样式生成。

### 关键要点
1. **什么是 Headless UI**
   - 只提供行为和可访问性
   - 不提供样式
   - 关注点分离

2. **Radix UI 的 Composition 模式**
   - 组合而非单一组件
   - AI 如何理解这种结构

3. **可访问性原语**
   - ARIA 属性自动注入
   - 键盘导航
   - 屏幕阅读器支持

4. **shadcn/ui 如何基于 Radix 构建**
   - Radix 原语 + Tailwind 样式

5. **横向对比**
   - Headless UI、React Aria、Ark UI 对比表

### 演示案例

## 课程总结

### 核心技术栈

**样式层**: Tailwind CSS v4
**组件层**: shadcn/ui + Radix UI
**设计工具**: Figma AI、v0.dev、Bolt.new、Penpot、Pencil.dev
**架构层**: Turborepo + pnpm Monorepo
**测试层**: Playwright MCP
**开发工具**: Cursor、Windsurf、Copilot、Claude Code
**AI 集成**: Vercel AI SDK
**代码质量**: Biome、TypeScript 严格模式
**全栈化**: Next.js Server Actions、tRPC、Prisma

### 核心原则

1. **语义内联**: Tailwind（样式在 HTML 中）
2. **源码可见**: shadcn/ui（代码在项目中）
3. **组合优于继承**: Radix 的 Composition 模式
4. **约定优于配置**: Turborepo、pnpm workspace
5. **AI 工具生态**: 完整的工具链支持

### 关键收获

1. **认知升级**: AI 友好性是新的技术选型维度
2. **技术栈重选**: 从传统技术栈迁移到 AI-Native 技术栈
3. **工作流重构**: 从传统开发流程到 AI 驱动的工作流
4. **效率提升**: 开发效率提升 3-5 倍
5. **职业发展**: 掌握 AI 时代的核心竞争力
